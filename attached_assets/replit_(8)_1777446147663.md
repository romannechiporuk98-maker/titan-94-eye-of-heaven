**Останнє оновлення:** 07.04.2026 | **Solo Deo Subjectus**

## 🎯 Що це за проект

**TITAN-94** — Telegram Mini App + Web Dashboard для моніторингу безпеки TON блокчейну.

- Аналіз вразливостей смарт-контрактів (Gemini AI)
- DEX арбітраж-сканер (DeDust, STON.fi, Megaton)
- Трекер конкурентів
- TON гаманець Reserve Fund
- AI самонавчання (знання зберігаються в PostgreSQL)
- Монетизація через TON-підписки (Free / Pro 5 TON / Elite 20 TON)
- Telegram Bot (aiogram 3)

---

## 🏗️ Архітектура (pnpm monorepo)

```
artifacts/
  api-server/          → Express 5 API (порт 8080)
  titan-dashboard/     → React + Vite Dashboard (порт 5000)
  titan-mobile/        → Expo React Native (scaffold)
  mockup-sandbox/      → Vite для canvas-компонентів

lib/
  db/                  → PostgreSQL + Drizzle ORM (5 таблиць)
  api-spec/            → OpenAPI spec + Orval codegen
  api-client-react/    → React Query hooks (auto-generated)
  api-zod/             → Zod schemas (auto-generated)

scripts/               → post-merge скрипти
```

## 🚀 Кроки для розгортання в новому акаунті

### 1. Завантажити архів
Завантажити `Titan-Neural-Hub.zip` і залити його в новий Replit (Node.js 24).

### 2. Розпакувати проект
```bash
unzip Titan-Neural-Hub.zip
# Скопіювати файли в корінь:
cp -r Titan-Neural-Hub/artifacts .
cp -r Titan-Neural-Hub/lib .
cp -r Titan-Neural-Hub/scripts .
cp -r Titan-Neural-Hub/attached_assets .
cp Titan-Neural-Hub/package.json .
cp Titan-Neural-Hub/pnpm-lock.yaml .
cp Titan-Neural-Hub/pnpm-workspace.yaml .
cp Titan-Neural-Hub/tsconfig*.json .
cp Titan-Neural-Hub/.gitignore .
cp Titan-Neural-Hub/.npmrc .
```

### 3. Встановити модуль Node.js 24
Через Replit Packages → встановити `nodejs-24`.

### 4. Встановити залежності
```bash
pnpm install
```

### 5. Додати PostgreSQL
Replit → Database → Add PostgreSQL → автоматично з'явиться `DATABASE_URL`.

### 6. Додати решту Secrets
Replit → Secrets → додати: `SESSION_SECRET`, `TELEGRAM_BOT_TOKEN`, `GEMINI_API_KEY`, `TON_API_KEY`.

### 7. Перенести схему БД
```bash
pnpm --filter @workspace/db run push
```

### 8. Налаштувати Workflows

**Workflow 1 — API Server (console):**
```
Name: API Server
Command: PORT=8080 pnpm --filter @workspace/api-server run dev
Wait for port: 8080
Output: console
```

**Workflow 2 — Dashboard (webview):**
```
Name: Start application
Command: PORT=5000 BASE_PATH=/ pnpm --filter @workspace/titan-dashboard run dev
Wait for port: 5000
Output: webview
```

## 🔐 Безпека — Telegram HMAC-SHA256

- Middleware `artifacts/api-server/src/middlewares/telegram-auth.ts`
- Кожен запит читає заголовок `X-Telegram-Init-Data`
- Верифікація через `crypto.createHmac("sha256", botToken)`
- Якщо initData відсутній — запит дозволяється (публічний доступ)
- `requireTelegramAuth()` — захист конкретних роутів
- Dashboard: `useTelegramWebApp.ts` → `setTelegramInitData(tg.initData)` при старті

## ❌ Що ще НЕ зроблено (станом на 07.04.2026)

### 🔴 КРИТИЧНО

#### 1. DB Schema mismatch — потрібен фікс!

**Проблема:** Роути `monetization.ts` та `ai-evolution.ts` вставляють дані БЕЗ обов'язкових полів:
- `subscriptions`: вставка без `externalId` і `planName` → помилка при першій підписці
- `knowledge`: вставка без `externalId` і `source` → помилка при AI trigger
- `subscriptions.expiresAt`: NOT NULL в схемі, але Free план передає `null`
- `subscriptions.amountPaidTon`: `real` в схемі, роут передає `toString()`
- `subscriptions.updatedAt`: роут використовує, але поля немає в схемі

**Фікс — змінити схему БД** (`lib/db/src/schema/subscriptions.ts`):
```typescript
// Зробити expiresAt nullable:
expiresAt: timestamp("expires_at"),  // видалити .notNull()

// Видалити planName або зробити nullable:
planName: text("plan_name"),  // видалити .notNull()

// Видалити externalId або генерувати auto:
// Використовувати uuid замість manual string
```

**Або фікс роутів** (`artifacts/api-server/src/routes/monetization.ts`):
```typescript
import { randomUUID } from "crypto";
// При insert додавати:
externalId: randomUUID(),
planName: plan.name,
expiresAt: plan.id === "free" ? new Date("2099-01-01") : expiresAt,
amountPaidTon: plan.price,  // number, не string
// Видалити updatedAt з onConflictDoUpdate
```

Аналогічно для `ai-evolution.ts` knowledge insert.

**Після фіксу:**
```bash
pnpm --filter @workspace/db run push
```
